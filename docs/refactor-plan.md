## 0. 목적 · 범위 · 원칙

**목적**: 분석으로 확정된 4개 결함 + 정리 1건을, 기존 v3 성과(터널 조립·분해, 노드 정합, 전역 레인)를 **회귀 없이** 개선한다.

**범위**
- 포함: 진입부 삭제 / 터널 전환 무이음 / 코어("핵") 중앙 / Career 그래프 중앙 / stale 주석.
- 제외: 신규 기능, 접근성(`prefers-reduced-motion`), `Math.random` 사전결함, 콘텐츠/CTA(별도 트랙).

**원칙**
1. **항목당 독립 커밋** — 롤백 단위 분리.
2. **측정 우선** — P2 2건은 "정적상 중앙"이라 **DOM/런타임 측정으로 진성 오프셋 확정 후** 보정.
3. **회귀 가드** — v3 #1(조립/분해)·#5(노드-선 정합)·#2(전역 레인)을 깨지 않는다.
4. 각 단계 후 `vite build` + `eslint`(기존 `Math.random` 3건 제외) + **양방향 스크롤 실측**.

### [P3] 정리 — `App.jsx` stale 주석

- 현재 주석: "busy 캔버스 씬은 bg-black으로 가려진다" — 그러나 이후 busy 씬 `bg-black`을 제거했으므로 **사실과 불일치**.
- 조치: 주석을 현재 동작(전역 레인이 busy 씬에도 노출)에 맞게 정정. (P0에서 App.jsx 손대므로 **함께 처리**)

## 2. 실행 순서 & 의존성

```
P0 (진입부 삭제)
  └─ App.jsx 변경 → P3(주석 정정) 함께 처리
        ↓ (스크롤 구조 −250vh 확정 후)
P1 (터널→코어칩 전환)   ← 별도 브랜치 권장, 리스크 大
        ↓
P2-a (핵 중앙)  →  P2-b (Career 중앙)   ← 독립, 병렬 가능
```

- **P0 먼저**: 스크롤 구조가 바뀌므로 P1 전에 확정.
- **P1은 격리**: 가장 위험 → 별도 브랜치/커밋.
- **P2 2건은 독립**: 측정 → 보정. 서로 영향 없음.
